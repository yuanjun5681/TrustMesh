> 版本：v1.0
> 日期：2026-04-07
> 作者：TrustMesh 产品设计

---

## 一、模块定位

**工作岗位市场**是 TrustMesh 平台的一个核心功能模块，旨在帮助用户快速获得"靠谱团队"，降低智能体使用门槛和企业部署成本。

用户不需要从零配置智能体，直接从岗位市场招聘现成角色，下载到本地即可组建团队开工。

---

## 二、核心概念

### 2.1 分布式协作模型

市场提供的是"岗位技能包"（角色定义文件），**不是在线服务**。

```
用户下载角色包
    ↓
本地 OpenClaw 加载角色（SOUL.md + AGENTS.md + IDENTITY.md）
    ↓
Clawsynapse 网络连接
    ↓
智能体作为 TrustMesh 团队成员参与协作
```

**关键理解：**
- 智能体运行在用户**本地机器**上
- Clawsynapse 是连接本地智能体到 TrustMesh 的网络底座
- 市场卖的是"角色定义"，不是"在线服务"

### 2.2 角色包结构

来自 agency-agents-zh 的 193 个预置角色，每个角色包含三个文件：

| 文件 | 内容 | 说明 |
|------|------|------|
| SOUL.md | 身份、人设、记忆、沟通风格 | 智能体的"灵魂" |
| AGENTS.md | 核心使命、技术交付物、工作流程 | 业务能力定义 |
| IDENTITY.md | 名称与简介 | 智能体身份标识 |

## 三、产品架构

### 3.1 用户操作流程

```
1. 浏览工作岗位市场
      ↓
2. 查看角色详情（职责、技能、使用场景）
      ↓
3. 点击下载或"招聘"按钮
      ↓
4. 下载或安装角色包（.zip 或压缩包）
      ↓
5. 按指引安装到本地 OpenClaw
      ↓
6. 重启 OpenClaw，角色自动激活
      ↓
7. Clawsynapse 自动将本地智能体接入 TrustMesh
      ↓
8. 智能体作为团队成员，可被 @ 调用参与工作
```

## 四、功能模块设计

### 4.1 工作岗位市场浏览

**入口位置：** TrustMesh 平台导航栏 → "工作岗位市场"

**页面布局：**
- 左侧：分类导航（按部门筛选）
- 右侧：角色卡片列表

**角色卡片内容：**
- 角色名称 + emoji
- 一句话简介
- 所属部门标签
- 关键技能标签（最多5个）
- "查看详情" 按钮

**搜索功能：**
- 支持按角色名称、部门、技能关键词搜索
- 支持模糊匹配

### 4.2 角色详情页

**页面内容：**

| 区块 | 内容 |
|------|------|
| 角色概览 | 名称、emoji、部门、一句话描述 |
| 核心职责 | 列出该角色的主要工作职责 |
| 专业技能 | 该角色擅长的技能领域 |
| 适用场景 | 适合在哪些项目/任务中使用 |
| 使用示例 | 典型的使用方式描述 |
| 角色预览 | SOUL.md / AGENTS.md / IDENTITY.md 内容预览 |
| 安装要求 | 对本地 OpenClaw 的版本要求等 |
| 操作按钮 | [招聘] 按钮 |

### 4.3 招聘流程（下载 + 安装）

**步骤一：点击"招聘"**
- 弹出下载或安装指引页面

**步骤二：下载角色包**
- 提供 .zip 压缩包下载（包含 SOUL.md + AGENTS.md + IDENTITY.md）
- 包含 README.md 安装说明

**步骤三：安装到本地**
```
1. 解压压缩包
2. 将角色文件放入 ~/.openclaw/agents/ 目录
3. 重启 OpenClaw 网关
4. 智能体自动出现在 Agent 列表中
```

**步骤四：接入团队**
- 本地 OpenClaw 启动后，通过 Clawsynapse 自动连接
- 在 TrustMesh 项目团队中添加该智能体为成员
- 智能体即可参与团队协作

### 4.4 后续扩展功能（第二阶段）

**用户上传角色：**
- 用户可创建自定义角色包
- 上传至平台审核
- 审核通过后上线到市场

**评分与评论：**
- 用户可对已使用的角色评分（1-5星）
- 可留下使用评价

**热度排名：**
- 按使用量、评分、下载量展示热门角色

## 五、技术说明

### 5.1 角色包来源

初始版本直接使用 [agency-agents-zh](https://github.com/jnMetaCode/agency-agents-zh) 项目中的 193 个角色。

**文件下载：**
```bash
# 克隆仓库
git clone https://github.com/jnMetaCode/agency-agents-zh.git

# 进入目录
cd agency-agents-zh

# 转换为 OpenClaw 格式
./scripts/convert.sh --tool openclaw

# 打包角色包
cd integrations/openclaw
zip -r ~/Downloads/roles.zip .
```

### 5.2 Clawsynapse 复用

Clawsynapse 网络已支持节点注册和连接，无需额外开发。

**本地 OpenClaw 接入 TrustMesh 的流程：**
1. 用户本地 OpenClaw 启动
2. OpenClaw 通过 Clawsynapse 连接 NATS 服务器
3. 智能体作为节点出现在 TrustMesh 可用成员列表中
4. 项目管理员可邀请该智能体加入团队

### 5.3 数据存储

| 数据 | 存储位置 | 说明 |
|------|---------|------|
| 角色元数据 | TrustMesh 数据库 | 角色名称、部门、技能等 |
| 角色文件 | 对象存储/CDN | SOUL.md 等文件可下载 |
| 用户上传角色 | 审核队列 → 对象存储 | 后续扩展功能 |
